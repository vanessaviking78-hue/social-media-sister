FEATURE 1: DELETED PRESET CLEANUP
When a client preset is deleted from the Presets page, the system needs to properly clean up the orphaned data instead of leaving it as a "ghost" preset. Currently a deleted preset still appears in fuzzy matching dropdowns and elsewhere.

On preset delete, please:

Clear all Meta-related fields on the preset row (metaPageAccessToken, metaFacebookPageId, metaInstagramAccountId, onboardingToken, clientPortalToken). If hard-deleting the row entirely, ensure these tokens are properly nulled first so no cached references remain.

Delete (or soft-delete) all Content Library items associated with this preset.

Delete all PENDING scheduledPostsTable rows for this preset (status = "scheduled" or "draft"). KEEP historical rows (status = "published" or "failed") for analytics and audit trail.

Remove the preset from any fuzzy-match suggestion logic during Bulk Connect (it should not be returned in the list of available presets if it has been deleted).

Show a confirmation modal before delete: "Delete [Preset Name]? This will permanently clear all unscheduled posts, library items and Meta connections for this client. Posted history will be kept for records. Type DELETE to confirm."

After successful delete, refresh the Presets page list so the deleted preset disappears immediately.


FEATURE 2: ENGAGEMENT AUTOMATION
Three sub-features that turn the CyberSuite into an engagement engine.
2A. Auto First Comment
When a post publishes to Instagram, automatically post a pre-written first comment 30-60 seconds later.

On every post creator (Carousel, Single Image, Reels, Stories, About Me, Seamless Carousel), add a new optional field "First comment" under the caption field. If left blank, fall back to the client preset's default first comment template.

Defaults per post type, editable per client preset:

Carousel: "Which slide surprised you most? Comment the number below 👇"
Single Image: "Save this one for later 💗"
Reel: "Drop a 🔥 if this helped"
Story: (no first comment, stories don't support comments)
About Me: "Tap a heart if you've felt this way too"

Database additions:

On post tables: firstComment: text, firstCommentPostedAt: timestamp
On clientPresetsTable: defaultFirstCommentCarousel, defaultFirstCommentSingle, defaultFirstCommentReel

New route in meta.ts:

POST /api/meta/post-first-comment — body: { presetId, igMediaId, commentText }. Calls Meta Graph API POST /{ig-media-id}/comments with message=commentText.

After the main post publishes and IG media ID is returned by Meta, wait 30 seconds, then call the first comment endpoint. Use setTimeout or a small queue, don't block the main publish response.

Required Meta permission: add instagram_manage_comments to existing scope list in meta-auth.ts. Surface a "Reconnect to enable" banner on the Presets page for users with old tokens.
2B. DM Keyword Auto-Responder
A native DM auto-responder inside the CyberSuite that replaces ManyChat. When someone DMs a client's Instagram with a specific keyword (e.g. "POLY"), the system auto-replies with a pre-written message.

New page in sidebar: DM Automations. Per client preset, user creates unlimited keyword-trigger rules:

Trigger keywords (comma-separated, case-insensitive, partial match)
Auto-reply message (rich text)
Optional link button
Optional image attachment
Active/paused toggle
Hit count display per rule

Database tables:

dmAutomationsTable:

  id, clientName, triggerKeywords (jsonb array), replyText, replyLinkUrl,

  replyLinkLabel, replyAttachmentUrl, active, triggerCount, lastTriggeredAt, createdAt

dmInteractionsTable (audit trail):

  id, automationId, senderIgId, senderUsername, incomingMessage, sentReplyText, firedAt

Backend webhook:

Set up Meta Webhooks subscription for instagram → messages
Route: POST /api/webhooks/instagram receives Meta's POST when a DM arrives
Verify webhook signature using META_WEBHOOK_VERIFY_TOKEN (Replit Secret)
Parse the message, find linked client preset by Page ID, check against active automations
On keyword match, send reply via POST /{page-id}/messages with recipient and message body
Log to dmInteractionsTable

Public webhook URL: https://thecybersuite.com/api/webhooks/instagram

Required Meta permissions: add instagram_manage_messages and pages_messaging to scope list.

Compliance: every auto-reply must end with an opt-out line: "Reply STOP at any time to opt out." If user replies STOP, log it and stop auto-responding to that sender.
2C. Optimal Post-Time Predictor
Analyses each client's audience activity insights and recommends off-peak slots where their followers are active but competition is low.

On the Content Calendar page, add a button: "Suggest optimal times for [client]".

Click triggers:

POST /api/scheduler/analyse-optimal-times with { presetId }
Server calls Meta Graph API GET /{ig-user-id}/insights?metric=online_followers&period=lifetime
Algorithm:
Identify top 30% of hours by follower activity
EXCLUDE standard round-hour times (9:00, 12:00, 17:00, 18:00, 19:00, 20:00) where competition is highest
Pick next-best off-peak slots with non-round minutes (:15, :22, :42, :47)
Spread across days of week (don't cluster on Mondays)
Return 5-7 slots formatted as { dayOfWeek, time }
Cache result on the preset for 7 days, show "Refresh recommendations" button

Database addition on clientPresetsTable:

optimalPostTimes: jsonb
optimalPostTimesUpdatedAt: timestamp

Required Meta permission: instagram_manage_insights.

UI: modal with a small bar chart visualisation of follower activity per hour, plus the 5-7 recommended slots with "Apply to next 14 unscheduled posts" button.
