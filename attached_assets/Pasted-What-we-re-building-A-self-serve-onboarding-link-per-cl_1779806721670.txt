What we're building
A self-serve onboarding link per client. Vanessa generates a unique link for each clinic preset, sends it to the clinic owner, they click it, log into their OWN Facebook, tick their Page, approve, and the token saves to Vanessa's preset for them. Vanessa never needs to be a Facebook Page admin again.

This solves two problems at once:

Connecting the clinics where Vanessa isn't a Facebook Page admin (currently ~25)
Onboarding all future new clients in 2 minutes flat

Note: email notification is deferred. Build everything below WITHOUT the email piece. We'll add Resend integration in a separate brief in 2 days.


User flow (clinic side)
Clinic owner gets a link from Vanessa via email or text, e.g. https://app.example.com/onboard/abc123token
Clicks it, lands on a branded Social Media Sister page (see Design spec below)
Sees the welcome message with their clinic name
Clicks "Connect Facebook & Instagram" button
Facebook OAuth opens in a new tab
They log into THEIR Facebook account, approve the app, tick THEIR Page(s)
Continue
Land back on the Social Media Sister page: "All done! Vanessa now has access to post for [Clinic Name]. You can close this tab."
User flow (Vanessa side)
On the Presets page, each preset has a new button: "Generate Onboarding Link"
Click it, server creates a unique reusable token, saves to preset
Link is shown with a Copy button: https://app.example.com/onboard/abc123token
Vanessa copies and sends to the clinic
Preset status flips to Connected, Page Access Token and Instagram Account ID populate automatically once the clinic completes
(Email notification will be added in a later brief)

Database changes
Add columns to clientPresetsTable:

onboardingToken text (nullable, unique index)
onboardingConnectedAt timestamp (nullable, set when client completes OAuth)

No expiry. Token is reusable forever. Regenerating creates a new token and invalidates the old one.


Backend implementation
New routes in artifacts/api-server/src/routes/onboarding.ts
POST /api/presets/:id/generate-onboarding-token

Generates a random token: crypto.randomBytes(24).toString("hex")
Saves to preset's onboardingToken column
Returns: { token, link: "${APP_BASE_URL}/onboard/${token}" }

GET /api/onboarding/:token

Looks up preset by token
Returns: { clinicName, alreadyConnected: boolean, connectedPageName: string | null }
Used by the public landing page to render clinic-specific welcome message and current state

GET /api/onboarding/:token/start

Looks up preset by token, builds OAuth URL with state encoding { mode: "onboarding", token }
Returns the URL OR redirects directly (frontend opens in new tab)
Scopes: instagram_basic,instagram_content_publish,pages_read_engagement,pages_show_list
Extend meta-auth.ts callback
After decoding state, if state.mode === "onboarding":

Look up preset by token (from state)
Exchange code for long-lived token, fetch pages (same as bulk flow)
If only ONE Page granted, auto-save to that preset (set metaPageAccessToken, metaFacebookPageId, metaInstagramAccountId, onboardingConnectedAt = now)
If multiple Pages granted, redirect to /onboard/:token/choose-page?key=... with a page selector specific to this onboarding flow
Redirect them to /onboard/:token/success


Frontend implementation
New public route: /onboard/:token
Routed OUTSIDE the authenticated app shell (no login required for clinics)
Single-page React component
Calls GET /api/onboarding/:token on mount, renders the appropriate state
Connect button calls window.open(/api/onboarding/:token/start) in a new tab
Polls /api/onboarding/:token every few seconds while OAuth tab is open, to detect completion
On detected completion, transitions to success state
New public route: /onboard/:token/choose-page
Page selector for clinics with multiple Pages (rare for clinic owners, but covers the case).
New section on each Preset card (existing Presets page)
Add a collapsible "Client Onboarding" section to the preset edit form:

If no token yet: button "Generate Onboarding Link"
If token exists: shows the link in a read-only text field with a Copy button + a "Regenerate (invalidates old)" link
Status pill: "Not connected yet" or "Connected on [date]"

Security notes
Anyone with the link can grant access for that specific preset. That's by design, it's how invite links work.
Facebook itself is the gatekeeper, so the client has to log into THEIR account.
Regenerating the token invalidates the old one immediately.
Recommend Vanessa sends links via email or DM, not by sharing publicly.

Clinic sees "All done!" success message
